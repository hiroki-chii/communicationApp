# 社内ランチ交流会マッチングシステム 運用マニュアル

本システムは、Google Apps Script (GAS) をバックエンドとし、スプレッドシートをデータベースとして活用する、社内コミュニケーション活性化のためのWebアプリケーションです。Gemini AIによる趣味・価値観を考慮したインテリジェントなグループマッチング機能、メンバー詳細ポップオーバー、リアルタイムグループチャット機能、およびセキュアな管理者向け管理機能を搭載しています。

---

## 1. システム概要とアーキテクチャ

### 1.1 全体アーキテクチャ
*   **フロントエンド**: シングルページアプリケーション (SPA) 構成、レスポンシブデザイン（PC・モバイル両対応）、アースカラー（オレンジ＆グリーン）のブランドデザイン、ライト/ダークモード対応。
*   **バックエンド**: Google Apps Script (main.gs)
*   **データベース**: Google スプレッドシート（「メンバー一覧」「設定」「部署マスタ」「マッチング履歴」などのシート）
*   **マッチングエンジン**: Gemini APIによるセマンティックマッチング ＆ 独自高速山登り法アルゴリズムのハイブリッド構成。

---

## 2. ユーザー権限とアクセス制御

### 2.1 権限の種類
システムには「一般ユーザー」と「管理者」の2つの権限が存在します。ログイン情報には、アクセスしたGoogleアカウントのメールアドレス (`Session.getActiveUser().getEmail()`) が自動的に使用されます。

*   **一般ユーザー**:
    *   システムへログインした全社員が該当します。
    *   次回マッチング結果の閲覧、「自分を探す」検索機能、グループチャット、自身のプロフィールの自己登録・更新が可能です。
*   **管理者**:
    *   スプレッドシートの「設定」シート内の `admin_emails` に登録されているユーザーです。
    *   システム初期化時に、最初に実行したユーザー（ヒロキ）が自動的に初代管理者に設定されます。
    *   一般ユーザー機能に加え、管理者ダッシュボード、AIマッチング生成・微調整、全メンバー管理（代理編集）、システム設定（APIキー、管理者追加）へアクセスできます。

## 3. 一般ユーザー向け機能と操作方法

### 3.1 次回マッチング確認（デフォルト画面）
ログイン後、最初に表示されるメイン画面です。最新の確定したマッチング結果（グループ分け）がカード形式で美しく表示されます。
1.  **グループ情報の閲覧**:
    *   各グループカードには、Gemini AIがメンバーの趣味や価値観を分析して命名した「ユニークなグループ名」と、会話を盛り上げるための「おすすめ雑談テーマ」が表示されます。
2.  **メンバー詳細ポップオーバー**:
    *   メンバーの名前にホバー（PC）またはタップ（スマホ）すると、スリムなポップオーバーが表示され、所属部署や「趣味」を確認できます。
3.  **「自分を探す」検索機能**:
    *   画面上部の検索窓に自分の名前やメールアドレスを入力すると、自分が所属しているグループカードがオレンジとグリーンのブランドグラデーションで強調表示（フォーカス）され、即座に自分のグループを見つけることができます。

### 3.2 プロフィール登録・変更
社員が自身のプロフィール情報を登録・更新する画面です。
*   **情報の自動読み込み**:
    *   自身のメールアドレスを入力すると、既に登録されているプロフィール情報（名前、部署、趣味、価値観等）が自動的にフォームにロードされます。
*   **入力項目**:
    *   **名前** / **所属部署**（部署リストから選択。「その他」を選択した場合は自由記述が可能）
    *   **参加形式**: 「対面」「リモート」「どちらでも」から選択できます。
    *   **趣味 / 特技 / 弱点**: それぞれ**全角25文字以内**で入力します。リアルタイム文字数カウンターが文字数を表示し、上限に近づくと警告色（黄→赤）に変化します。
    *   **仕事・人生の価値観（5段階評価）**: 「やりがい」「チームワーク」「プライベート」「評価」「成長」の5項目について、自身が重視する度合いを1（低）〜5（高）で選択します。
    *   **ステータス（アクティブ / 非アクティブ）**: 次回ランチ会に参加可能かどうかを切り替えます。非アクティブに設定すると、マッチング対象から除外されます。
    *   **次回優先フラグ**: 次回ランチ会に「優先的に参加したい」場合にチェックを入れます。

### 3.3 グループチャット機能
マッチング確定後、同じグループのメンバー同士でリアルタイムに連絡を取り合えるチャットスペースです。
*   対面ランチの場所決め、リモートランチのURL共有、日程調整に利用できます。
*   チャットヘッダー内のメンバーアイコン一覧から、各メンバーの「趣味」などをポップオーバーで手軽に確認できます。

## 4. 管理者向け機能と操作方法

管理者は、サイドバーまたはボトムナビゲーションから「管理者パネル」に切り替えることができます。

### 4.1 管理者ダッシュボード
*   現在の「登録メンバー総数」「次回参加予定のアクティブユーザー数」「過去のマッチング実施回数」などのサマリー数値を確認できます。

### 4.2 インテリジェントAIマッチングの生成
新規のマッチンググループを作成・確定する手順です。
1.  **パラメータの設定**:
    *   1グループあたりの目標人数（標準は4人）を指定します。
2.  **マッチング生成の実行**:
    *   「Gemini AIマッチング」または「山登り法（高速ロジック）」を選択してマッチングを生成します。
    *   Gemini AIマッチングでは、メンバーが登録した「趣味」「特技」「弱点」「仕事・人生の価値観（5段階スコア）」および「参加形式（対面/リモート）」の親和性を総合的に分析し、会話が最も弾む組み合わせを全自動で編み出します。
3.  **手動ドラッグ＆ドロップ調整**:
    *   生成された仮グループのメンバーを、管理者自身がマウス操作でドラッグ＆ドロップして他のグループへ移動させ、最終微調整を行うことができます。
4.  **マッチングの確定・公開**:
    *   「マッチングを確定する」ボタンを押すと、結果が「マッチング履歴」シートに保存され、一般ユーザー画面に最新結果として公開されます。

### 4.2.1 Geminiが参照・考慮するユーザー情報
Gemini AIによるマッチング実行時、バックエンドからGemini APIに対して、以下のユーザー情報がオブジェクト形式でセキュアに連携され、グループ選出および雑談テーマの考案に活用されます。

1.  **基本属性情報**:
    *   **メンバーID / 名前 / 所属部署 (dept)**: 部署の重複を避けて「部署の多様性」を最大化するために活用されます。
2.  **パーソナル属性 (自己紹介項目)**:
    *   **趣味 (interests)**: 共通の興味関心を持つメンバーを繋げ、会話の強力なフックを作ります。
    *   **特技 (specialty)**: メンバー個々の強みやユニークなスキルから、盛り上がるトピックを引き出します。
    *   **弱点 (weakness)**: 人間味のある自己開示による親近感の醸成、またはお互いにクスッと笑い合えるような雑談テーマの生成に活用されます。
3.  **仕事・人生の価値観 (values - 5段階スコア)**:
    *   「やりがい」「チームワーク」「プライベート」「評価」「成長」の5つについて、各スコアの親和性や価値観のバランスを多角的に分析し、居心地が良く刺激的な組み合わせを導き出します。
4.  **配慮事項 (considerations) の尊重と非開示制御**:
    *   アレルギーや時間制限などの配慮事項が登録されている場合、Geminiはそれらの要望を尊重したグループ分けを行います。
    *   **セキュリティとプライバシー保護**: デリケートな情報（アレルギーや健康状態など）の漏洩を防ぐため、Geminiが自動作成する全体公開用の紹介文（`memo`）には、**「配慮事項」に関する具体的な内容は一切出力しないようにシステムインストラクションによって厳格に制御**されています。

### 4.3 メンバー管理（マスタメンテナンス）
登録されている全メンバーの一覧をテーブル形式で管理します。
*   **価値観のカラーバッジ可視化**:
    *   メンバーの価値観5項目のスコアが、5（エメラルド）、4（スカイブルー）、3（スレートグレー）、2（アンバー）、1（ローズ）の濃いソリッドカラーバッジで表示され、視覚的に傾向を把握できます。
*   **参加ステータス・次回優先の即時トグル**:
    *   テーブル上のチェックボックスをクリックするだけで、サーバーサイドのスプレッドシート値が瞬時に更新され、日本語でわかりやすい完了トーストが表示されます。
*   **代理登録・編集モーダル**:
    *   管理者は、メンバーの「新規追加」や「登録情報の編集」を代理で行うことができます。一般画面と同様の文字数バリデーションや部署選択ロジック（「その他」対応）が完全に適用されます。
*   **一括削除・検索フィルター**:
    *   部署（「その他」含む）や名前、メールアドレスによる瞬時のテーブル内検索フィルタリングが可能です。

### 4.4 システム設定
*   **管理者リストの追加・編集**: カンマ区切りで管理者メールアドレスを編集・更新できます。
*   **Gemini APIキーの設定**: Gemini APIの接続検証テストを行えます。
*   **データベース初期化（リセット）**: 初期状態に戻したり、テスト用デモメンバーを一括で自動生成することができます。

## 5. 運用上の注意点 ＆ トラブルシューティング

### 5.1 スプレッドシートの直接編集について
*   **注意**: スプレッドシートのヘッダー名（1行目）や列順を、手動で直接変更しないでください。プログラムの読み書きインデックスとズレが生じ、システムエラーの原因となります。
*   **カラム構成の自動修復機能**:
    *   本システムには自動マイグレーションが実装されています。万が一、「参加形式」や「特技」「弱点」「価値観」などの新設列が存在しない場合、アプリ起動時（`initDatabase` 実行時）に安全にデータを退避させつつ、正しい16列構成へリビルド・マイグレーションを行い、不正なチェックボックスデータ等の自動クレンジングを行います。

### 5.2 アプリの更新・デプロイ方法（管理者向け）
プログラムの修正を反映させる手順です。
1.  **claspを用いたプッシュ**:
    *   ファイルの修正後、以下のコマンドを使用してGoogle Apps Scriptへコードをプッシュします。
    ```bash
    clasp push
    ```
2.  **Web Appのデプロイバージョンの更新**:
    *   スプレッドシートのメニューから `拡張機能 -> Apps Script` を開きます。
    *   右上にある `デプロイ -> デプロイの管理` をクリックします。
    *   有効なWeb Appデプロイの編集（鉛筆マーク）をクリックし、バージョンを「新バージョン」として保存します。
    *   **重要**: アクセスできるユーザーが「全員（匿名を含む、または組織内）」、実行ユーザーが「アクセスしているユーザー」または「自分（開発者）」の要件に合わせて適切に選択されていることを確認してください。

### 5.3 よくある質問 (FAQ)
*   **Q. 「その他」の部署として登録されたメンバーはどこに分類されますか？**
    *   A. 管理者画面の部署絞り込みで「その他」を選択すると、部署マスタに登録されていない自由記述の部署や、テキストが「その他」のメンバーが一発で抽出されます。
*   **Q. 趣味・特技・弱点が25文字を超えて登録できないのはなぜですか？**
    *   A. ポップオーバーやグループカードなどの限られたUI領域で文字がはみ出さず、美しくコンパクトに収めるための仕様です。
